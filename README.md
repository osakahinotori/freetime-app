# 空き時間メーカー

Googleカレンダーの予定から「空き時間」を計算し、おしゃれで見やすいカレンダー画像（400dpi PNG）として書き出すWebアプリです。スマホのブラウザだけで完結し、ホーム画面に追加すればアプリのように使えます。

- 予定データはあなたの端末内（ブラウザ）だけで処理され、外部サーバーには送信されません。
- Googleへの接続は **読み取り専用** スコープのみ使用します。

## できること
- 開始日時・終了日時で表示する日程範囲を指定
- アカウント内の複数カレンダーから、含めるものをその都度チェックで選択
- 「空き時間としない時間帯」（夜間など）をその都度設定（日跨ぎ可）
- 最小の空き時間（30分以上など）でフィルタ
- グリッド＋リストのハイブリッド画像を 400dpi PNG で保存／共有

---

## セットアップ（最初の1回だけ）

Googleカレンダーを読むには、あなた自身の **OAuth クライアントID** が必要です。

### 1. Google Cloud でプロジェクトとOAuthクライアントを作る
1. https://console.cloud.google.com/ にアクセス
2. プロジェクトを作成（任意の名前）
3. 「APIとサービス」→「ライブラリ」で **Google Calendar API** を有効化
4. 「APIとサービス」→「OAuth同意画面」を構成
   - User Type: **外部**
   - アプリ名・サポートメール等を入力
   - 「対象ユーザー」に **自分のGoogleアカウントをテストユーザーとして追加**
     （テスト中はここに追加したアカウントだけが使えます）
5. 「認証情報」→「認証情報を作成」→ **OAuth クライアントID**
   - アプリケーションの種類: **ウェブアプリケーション**
   - **承認済みの JavaScript 生成元** に、アプリを公開するURLを追加
     - ローカル確認用: `http://localhost:8000`
     - 本番用: 後述のデプロイ先URL（例 `https://your-name.netlify.app`）
   - リダイレクトURIは不要です（このアプリはトークン方式を使用）

### 2. クライアントIDを設定
発行された `xxxxx.apps.googleusercontent.com` を `config.js` に貼り付けます。

```js
window.APP_CONFIG = {
  GOOGLE_CLIENT_ID: "xxxxx.apps.googleusercontent.com",
};
```

---

## ローカルで動かす（Mac）
このフォルダで簡易サーバーを起動します（`file://` では Google 認証が動きません）。

```bash
cd ~/freetime-app
python3 -m http.server 8000
```

ブラウザで http://localhost:8000 を開く。

> デモ確認: 接続前でも、ブラウザの開発者コンソールで `__demo()` を実行するとサンプルデータで画像描画を確認できます。

---

## スマホで使う（デプロイ）
Google OAuth は https が必須なので、無料の静的ホスティングに置きます。どれか1つでOK。

- **Netlify Drop**: https://app.netlify.com/drop にこのフォルダをドラッグ＆ドロップ
- **GitHub Pages**: リポジトリにpushして Pages を有効化
- **Vercel / Cloudflare Pages**: フォルダをそのままデプロイ

デプロイ後のURLを、手順1-5の「承認済みの JavaScript 生成元」に必ず追加してください。
スマホでそのURLを開き、Safari/Chrome の「ホーム画面に追加」でアプリ化できます。

---

## 使い方
1. 「Googleと接続」をタップ（初回のみ許可）
2. 日程範囲を設定（クイックボタンも利用可）
3. 含めるカレンダーにチェック
4. 空き時間としない時間帯を調整（既定は 22:00〜08:00）
5. 「空き時間を計算して画像化」→ プレビュー確認
6. 「画像を保存」または「共有…」で相手へ

---

## ファイル構成
| ファイル | 役割 |
|---|---|
| `index.html` | 画面 |
| `styles.css` | スタイル |
| `config.js` | OAuthクライアントID設定 |
| `app.js` | UI・Googleカレンダー連携 |
| `freebusy.js` | 空き時間の計算ロジック |
| `render.js` | カレンダー画像の描画 |
| `png-dpi.js` | PNGへの400dpi書き込み |
| `manifest.webmanifest` | PWA設定 |

## 仕組みのメモ
- 予定の取得は Calendar REST API（`calendarList`, `events`）を fetch で直接呼び出し。
- 「予定なし（transparent）」やキャンセル済みは busy に含めません。
- 空き = 日程範囲 −（除外時間帯）−（予定）。終日予定の日は任意で終日ビジー扱い。
- 400dpi は PNG の `pHYs` チャンクに物理解像度を書き込んで実現（印刷時に400dpiになります）。
